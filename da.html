Introduction

Data analysis is a process of inspecting, cleansing, transforming, and modelling data with the goal of discovering useful information, informing conclusions, and supporting decision-making. Data analysis plays a role in making decisions more scientific and helping businesses operate more effectively.

Data Wrangling

Data wrangling is the process of transforming data from a certain form into another format, with the intent of making it more appropriate and valuable for a variety of downstream purposes such as analytics.

On the titles dataset, we create a new job titles column with the format “specialization + rank, department” , exceptions to this rule are:

  • If department is Executive or starts contains the word Chief, use specialization + rank
  • If rank has the words Lead or Senior, these should prefix the job title
  • If specialization is Project Management, job title format is inverted i.e. starts with rank + specialization, department
  • If department contains the word Leadership, replace department with group in the job title 


q1 <- titles %>%
  mutate(nn = if_else(Department == "Executive" | str_detect(`Rank/Level`, "Chief"), str_c(str_replace_na(Specialisation), " ", `Rank/Level`),
                      if_else(str_detect(`Rank/Level`, "Lead"), str_c("Lead ", str_replace_na(Specialisation), " ", str_remove(`Rank/Level`, "Lead"), ", ", Department ),
                              if_else(str_detect(`Rank/Level`, "Senior"), str_c("Senior ", str_replace_na(Specialisation), " ", str_remove(`Rank/Level`, "Senior"), ", ", Department ),
                                      if_else(str_detect(Department, "Leadership") | str_detect(titles$Department, "Leadership") & is.na(titles$Specialisation), str_c(str_replace_na(Specialisation), " ", `Rank/Level`, ", ", Group), str_c(str_replace_na(Specialisation), " ", `Rank/Level`, ", ", Department))))))

q1$`Updated Job title` <- str_remove(q1$nn, "NA ")

Insights

Users may have particular data points of interest within a data set, as opposed to the general messaging outlined above. Such low-level user analytic activities are presented in the following table.

The best way to get insights from data is to ask yourself questions about the data, and see if the data can answer these questions. These questions can belong to different analytic techniques like:

  • Retrieve Value : Given a set of specific cases, find attributes of those cases ie. What are the values of attributes {X, Y, Z, …} in the data cases {A, B, C, …}? eg. How many active employees have a degree as their highest level of education?
  • Filter : Given some concrete conditions on attribute values, find data cases satisfying those conditions. ie. Which data cases satisfy conditions {A, B, C…}? eg. How many active teachers are there for each job title?
  • Compute Derived Value : Given a set of data cases, compute an aggregate numeric representation of those data cases. ie. What is the value of aggregation function F over a given set S of data cases? eg. What is the median age of male employees vs female?
  • Find Extremum : Find data cases possessing an extreme value of an attribute over its range within the data set. ie. What are the top/bottom N data cases with respect to attribute A? eg. On average, which academy has the youngest employees, which has the oldest?
  • Sort : Given a set of data cases, rank them according to some ordinal metric. ie. What is the sorted order of a set S of data cases according to their value of attribute A? eg. Which Base academies with the highest turnover of employees?

Conclusion

Performing effective data analysis involves obtaining relevant facts. These facts are then used to answer questions, support a conclusion or formal opinion, or to test hypotheses. Facts by definition are irrefutable, therefore, when analysis is done, there should be consensus around the facts.

When performing data analysis, one should be aware of cognitive biases that can adversely affect analysis. For example, confirmation bias which is the tendency to search for, or interpret information in a way that confirms one’s preconceptions.

Effective analysts should also be adept with a variety of mathematical techniques, since bad math techniques may produce misleading results or may misinform.

Finally, when communicating analysis results, one may implement a variety of data visualization techniques to help communicate the message more clearly and efficiently to the audience.
